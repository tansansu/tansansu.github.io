Hugo를 통해 생성한 페이지를 Github에 간단히 연결하기

28/Feb 2016
TIPS

최근 Github에 올리는 블로그 페이지들을 jekyll에서 Hugo로 바꿨습니다. 정적페이지 생성기로 jekyll 외에 여러가지가 있다는 것을 알게 되었고, Hugo는 우연히 인터넷에서 알게 되어 한 번 시도만 해볼까 했는데 결국에는 블로깅 Tool도 바꿔버렸습니다 ^^; 겉으로 보기에는 당연히도 차이점이 없지만 만족스러운 전환이였습니다.

제가 생각하는 Hogo의 장점은 > 1. 설치과정이 쉽다. 저는 mac에서 jekyll을 설치하고 사용했는데, jekyll을 설치하는 과정이 복잡하고 번거로웠습니다. 반면에, Hugo는 mac 기준으로 home brew를 설치했다면 brew install hugo 명령어만으로 모든 설치가 끝납니다. > 2. 페이지 생성 속도가 빠르다. Hugo가 Jekyll보다 확실히 페이지 빌드 속도가 빠릅니다. 포스팅이 20개도 안되는 제 블로그지만 Hugo로 빌드하는데 43ms 정도 걸리네요. jekyll에서는 1초 정도는 걸렸던 거 같습니다.

이 외에 jekyll 못지않게 매뉴얼이나 테마 등이 잘 되어 있습니다. 왠만한 테마는 Disqus 댓글 및 Google Analytics가 전부 내장되어 있습니다 ^^;

Hugo의 불편한 점

Hugo가 다 좋은데 한가지 번거로운 점이 있는데, Github의 블로그 계정에 연결하려면 추가 작업이 필요합니다. 빌드하게 되면 폴더 내에 public 이라는 별도의 폴더가 생성되고, 그 안에 웹페이지가 저장됩니다. 그래서 Hugo 폴더를 Github의 블로그 Repository에 바로 연결하면 웹페이지가 보이지 않습니다. public 폴더가 연결되야 하기 때문입니다.

그래서 Hugo의 홈페에지에는 Github의 subtree를 기능을 이용하라고 나와 있습니다. 그 외에 제가 처음 Hugo를 사용할 때 참고한 블로그 페이지에 나온 것처럼 쉘 스크립트를 쓰기도 하구요. 그런데 이것보다 더 간편하다고 생각되는 방법을 소개합니다.

Public 폴더를 Symlink로 연결하기

Hugo 폴더 밑에 생성되는 Public 폴더를 Github의 블로그 Repository와 연동되는 폴더로 Symbolic Link를 걸어주면 간단히 해결됩니다. (Symbolic Link와 관련된 자세한 설명은 웹에 많이 있기 때문에 자세한 설명은 생략하겠습니다)

  1. Mac에서 Symlink 생성하기 ln -s "타겟폴더 경로(Public 폴더 경로)" "Github와 연결할 폴더 경로" 예를 들면 이렇게 하면 됩니다. ln -s /Users/계정명/Hugo/public /Users/계정명/xxx.github.io (위의 “xxx.github.io”는 Github에 생성된 블로그 Repository명 입니다.)

  2. Windows에서 Symlink 생성하기 mklink /j "타겟폴더 경로(Public 폴더 경로)" "Github와 연결할 폴더 경로"

subtree 기능도 한 번 설정하면 간편해서 Symlink방법과 별차이가 없을 수 있는데요, subtree는 Hugo의 모든 파일이 일단 Github에 업로드 되기 때문에 블로그와 연관된 Repository가 중복으로 생성되는 점이 있으며, md파일로 생성한 블로그 포스트 raw 파일도 불필요하게 Github에 공개되는 점이 있습니다. 웹으로 생성된 페이지만 Github에 올리고 싶다면 Symlink 방식이 가장 간편한 것 같습니다. ^^
