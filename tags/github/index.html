Mac에서 설치가 안되는 R Library 설치하기

November 21, 2016

1 minute read

Mac에서 MRO(Microsoft R Open) 3.3.1을 쓰고 있다. 텍스트마이닝 작업을 위해서 tm 패키지를 설치하려고 했는데 도무지 설치가 되지 않았다. tm 패키지를 설치하려면 slam이라는 dependency를 설치해야 하는데, slam 패키지는 소스코드를 내려받아 컴파일해서 설치하는 방식이였다. 이 slam 패키지가 컴파일 하다가 오류가 나는지 아래 같은 에러가 나면서 도무지 설치가 되지 않았다. Mac에서 사용하는 clang 컴파일러가 문제인가 싶어서 gcc를 새로 설치하고 별짓을 다해도 안되다가, 3일을 구글을 뒤져서 방법을 찾아냈다. https://support.rstudio.com/hc/en-us/community/posts/222375268-slam-compile-question-on-Mac?sort_by=votes Some packages built for OS X require a ‘gfortran’ installation.
